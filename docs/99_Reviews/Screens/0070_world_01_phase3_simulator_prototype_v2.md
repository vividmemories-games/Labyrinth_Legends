## Task Summary

Built a **simulator-only** visual prototype for World 01 Forgotten Citadel that answers: *“Does this finally feel like a believable ancient place?”* Phase 1 (art direction) and Phase 2 (Flutter feasibility) guided a full-screen **2.5D environment compositor** replacing the prior wallpaper/card approach. The puzzle board is integrated as a carved courtyard floor — not a floating card — with foreground/midground/background depth, architectural scale (colonnade, gate, distant wall, broken pillars, braziers), warm sandstone materials, and a subordinate inscribed HUD. M2.7 gameplay on `level_003` remains fully playable.

**Explicitly out of scope (respected):** engine / `GameplaySession` / level-rule changes, M2.8 production work, world system, shaders-first approach, LLDS documentation updates for this phase, production claim.

## Motivation

**Why:** Phase 1 documented why reference boards work (world predates player, board is architecture, scale is emotional grammar). Phase 2 confirmed Flutter can approximate this via layered 2.5D composition — not wallpaper + pasted grid. The prior citadel experiment still read as “puzzle board on a background.”

**Problem solved:** Validates whether Flutter can deliver the Forgotten Citadel *feeling* inside the real app before committing to M2.8 production art pipeline.

**Approach chosen:** Isolated experiment module (`lib/features/gameplay/presentation/experiment/forgotten_citadel/`) gated to `level_003` only. Full-screen `Stack` compositor with static `CustomPainter` layers + `RepaintBoundary`, board positioned in environment with recess/AO/lip integration. Presentation wiring only — no gameplay logic in painters.
