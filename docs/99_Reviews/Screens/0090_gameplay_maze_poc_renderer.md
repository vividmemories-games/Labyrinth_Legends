## Task Summary

Human requested production Gameplay use the maze-render POC (`MazePainter` code-drawn raised stone walls) instead of the PNG tile pipeline (`tile_floor` + `edge_*` overlays). Delivered: promoted POC rendering into `lib/features/gameplay/presentation/rendering/`, rewired `BoardRenderer` to paint the maze via `MazePainter` with `CellOverlay` for interaction tints and collectible sprites, removed `TileView`/`CellEdgeMask`, trimmed `LLGameplayAssets` edge/tile kinds, deleted bundled edge/floor PNGs, migrated tests, and updated LLDS. Out of scope: world theme switching UI, Amazon Jungle theme in production, dev POC screen route, engine/schema changes, HUD sprite changes.

## Motivation

Authored edge overlay PNGs failed visual QA — opaque seams, inconsistent floors, unsustainable per-theme tileset production. The POC renderer draws neighbor-aware raised stone edges in code with optional seamless JPG textures, producing a seamless board at any resolution. Human explicitly preferred the POC look over the PNG autotile approach.
