## Task Summary

Implemented production-quality runtime world updates and objective feedback for M2.4. The board now renders engine-driven `session.grid` mutations (collected gems disappear from tiles). Collected keys are hidden from tiles via `context.collectedKeys`. The objective panel and resource display reflect live engine attempt context (gems collected, exit reached, terminal outcomes). Post-execution `restart()` was removed; execution now transitions to explicit `terminalWon` / `terminalLost` shell phases and remains there with lightweight completion cues — no victory screen, rewards, navigation, or automatic restart. Engine code unchanged.

**Out of scope (respected):** rewards, stars, level progression, victory/failure screens, particles, sound, animation polish, responsive changes.

## Motivation

**Why:** M2.3 executed paths but discarded terminal engine state via automatic restart. Players could not see the world react or objectives resolve.

**Problem solved:** The gameplay screen now exposes engine truth — runtime grid, objective progress, and terminal won/lost state.

**Approach:** Derive all presentation from `GameplaySession` (`grid`, `context`, `phase`, `statusMessage`). Controller maps engine terminal phases to shell phases only; no gameplay rules in UI.

### Runtime World Updates

- **Gems:** `session.grid` runtime mutation (`hasGem: false`) rendered by `TileView`
- **Keys:** Hidden when `cell.keyId` is in `session.context.collectedKeys`
- **Doors/switches:** Not rendered — engine does not mutate grid for these in M1.4 subset

### Objective Feedback

- `objectiveProgressLabel` derived from `session.context` + `level.objectives`
- Exit: `Exit pending` / `Exit reached` / `Objectives complete`
- Gems: `N collected · M remaining` or terminal engine message
- Resource display: `collected / initial` from session context

### Terminal State

```text
executing → executeNextStep loop
  → engine evaluates (won/lost)
  → shell terminalWon | terminalLost
  → remain in terminal (no restart, no navigation)
```

### Implementation Boundaries

- UI does not call `session.restart()` after execution
- No reward/stars UI even when `session.reward` is populated internally
- `executionComplete` shell phase retained for edge cases where engine phase is still `executing`
- Retry/replan action deferred to M2.5+
