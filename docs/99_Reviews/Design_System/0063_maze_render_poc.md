## Task Summary

Human Owner reported that the tile-image approach to maze rendering was failing: producing pixel-perfect autotile PNG sets (16–47 variants for one-/two-/three-side raised edges and corners) per world theme is not sustainable, and re-theming (e.g. Ancient Ruins → Amazon Jungle) would require redrawing the full set. This POC replaces tileset images with code-drawn geometry: all wall cells are merged into a single vector path with neighbor-aware corner rounding, extruded and shadowed for a raised 3D look, and skinned by a swappable `MazeTheme` (colors + optional seamless textures). Delivered as a dev-only screen at `/dev/maze-render-poc` with two example themes and AI-generated seamless textures. Out of scope: integration with the production gameplay screen, decoration props, LLDL formalization of theme tokens.

## Motivation

- Autotile PNG pipelines require 16–47 pixel-perfect, seam-matched tiles per theme; AI image generation is unreliable at seam-matching, and hand tuning per theme does not scale for a solo developer.
- Vector geometry drawn in `CustomPaint` is seam-free by construction, resolution independent, and derives every edge/corner variant automatically from the grid — the exact "one side / two sides / three sides raised" problem the tileset was trying to solve.
- Themes reduce to a small data bundle (palette + 2 seamless textures), so new worlds are cheap and existing worlds can be re-skinned without art rework.

Per Human Owner feedback (2026-07-07), the default wall style is `edge`:
raised edges drawn on tile boundaries wherever a walkable tile borders a
wall cell (`WallEdgeBuilder`), so walls are sides of tiles rather than
occupants of tiles, matching the ancient-ruins design reference. Wall-cell
regions render as slightly darkened fenced-off floor — natural anchor
spots for future decoration props. The original full-cell `block` style
remains available via the screen toggle.

Second feedback round (2026-07-07): edges are rendered as individual
jittered stone blocks (deterministic per-stone length/thickness/offset
variation, ~12% damaged stones with corner-safe length bites) with the
wall texture applied near-full-strength plus a per-theme multiply tint,
so walls read as old rugged masonry rather than clean vector bars.

> Renders produced through `MazePainter` off-screen at 700×700; identical code path to the dev screen.

## Known Issues

- Generated textures are near-seamless, not perfectly seamless; `TileMode.mirror` hides edges but can produce visible symmetry at large texture scales.
- Exit/start markers are placeholder shapes, not final art.
