## Task Summary

Created **WS11_Components_Language.md** — the permanent reusable components authority for LLDL. Thirty-two sections formalizing eight component principles (LLDL expressions, token inheritance, clarity before decoration, gameplay restraint, meta-game craft, multichannel states, accessibility by default, consistency over novelty). Covers inheritance model, functional vs ceremonial tiers, domain families (buttons, panels, modals, navigation, HUD, lists, rewards, artifacts, store, settings), states, feedback, motion, typography, iconography, accessibility, composition, governance, anti-patterns, and decision framework. Updated `LLDL.md` and WS10 navigation. **Out of scope:** exact APIs, dimensions, widget trees, code, production workflows.

## Motivation

**Why:** WS0–WS10 defined channel and token philosophies — but no document owned how reusable UI building blocks inherit, behave, and govern consistency across gameplay and meta-game.

**Problem solved:** Prevents generic platform widgets, one-off styled controls, HUD clutter, undifferentiated component sprawl, and screen-specific UI inventions that drift from LLDL.

**Approach:** WS11 capstone components workshop extending WS0–WS10; precedes `Components.md` reconciliation and screen spec component references.
