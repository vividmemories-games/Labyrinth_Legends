## Reviewer Notes

### Codex Engineering Review

- **Date:** 2026-07-03
- **Verdict:** Needs changes before approval
- **Score:** 8.0 / 10

**Blocking Issues:**

1. **`loadWorld` will fail the current level-select/app path**

   `pubspec.yaml` bundles the whole `assets/levels/world_1/` folder, and `LevelRepository.loadWorld` parses every JSON asset under that folder. This package already notes that levels 004–020 lack `schemaVersion`, so `loadWorld('world_1')` will throw as soon as it reaches `level_004.json`. Because `worldLevelsProvider` powers level loading, this is not only a known content issue; it blocks existing world-level consumers.

   **Required fix:** Migrate or remove invalid bundled levels, filter runtime world loading to the validated M1 band, or otherwise keep out-of-scope assets invisible to runtime `loadWorld`.

2. **Validator can still pass malformed schema fields that `fromJson` then crashes on**

   `Level_Format.md` requires load-time structural validation before parsing, but `LevelFormatValidator` does not validate several required or typed schema fields before `LevelDefinition.fromJson` and model constructors cast them. Examples include missing or non-string `name`, invalid cell modifier types such as `hasGem: "true"` or `keyId: 3`, invalid `visibility`, and malformed `tutorial` fields.

   **Required fix:** Extend structural validation so malformed authored JSON produces typed validation errors instead of `TypeError` or `FormatException` during parse.

**Non-blocking Issues:**

1. **Analyzer result in the review package is stale/inaccurate**

   The review package says warnings in the new validator test file were fixed, but `flutter analyze --no-fatal-infos` currently exits non-zero with three warnings in `test/game_engine/level_format_validator_test.dart`, plus existing infos.

   **Recommended fix:** Remove the unnecessary casts in the validator test file and update the reported analyzer result.
