## Task Summary

Implemented stateless `ObjectiveEvaluator` with `ObjectiveEvaluationResult` / `ObjectiveOutcome`, integrated into `GameplaySession.executeNextStep()` so the session applies `won` / `lost` lifecycle transitions after each execution step. M1.5 evaluates `reachExit`, `collectAllGems`, and `minGems` per Level_Format §13. **Out of scope:** `RewardCalculator`, star calculation, save/progress, UI/provider rewrite, hazards, new schema fields.

## Motivation

**Why:** M1.4 executed paths but never determined outcomes. M1.5 closes the core engine loop: execution facts → objective assessment → lifecycle transition per GP5 and EA-007.

**Problem solved:** `executeNextStep()` left sessions in `executing` after exit or failure; no GP5 completion evaluation existed.

**Approach:** Stateless `ObjectiveEvaluator` reports `continue` / `won` / `lost`; `GameplaySession` orchestrates evaluation after every step and owns phase transitions per EA-008.

## Objective Evaluation Behavior

### When evaluator runs

**After every `executeNextStep()`** — including the final movement step and idempotent path-complete calls.

Rationale: GP5 §11 ("During execution — per-step hazard/objective state updates") and GP7 step 8–9; task prompt preferred default. Loss transitions immediately when `attemptContext.failed` is true; win transitions as soon as all required objectives are satisfied.

### Evaluation order (precedence)

1. **Lost — execution failure** (`attemptContext.failed`) → immediate `lost` (GP7 §12: failure before completion)
2. **Won — all required objectives met** → `won` (may occur before path end if exit reached and gems satisfied)
3. **Lost — path complete with unmet objectives** → `lost` (objective impossibility / GP5 §12 — no silent soft-lock)
4. **Continue** — otherwise remain in `executing`

### Required objectives (M1 / schema v1)

| Objective | Rule |
|-----------|------|
| `reachExit` | `attemptContext.reachedExit == true` |
| `collectAllGems` | `gemsCollected >= level.grid.countGems()` |
| `minGems` | `gemsCollected >= minGems` when `collectAllGems` is false |
