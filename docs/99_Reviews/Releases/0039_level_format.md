**Why:** Technical Implementation Plan §14 identified `Level_Format.md` as P0 blocker for M1 engine coding. Existing level JSON files were authored without a formal schema, creating fragile loaders and content bugs.

**Problem solved:** Gives Codex and Cursor a single contract for level content vs engine behaviour — JSON describes **what** is on the grid; `game_engine/` implements GP2/GP5/GP7 rules.

**Approach:** Synthesize existing `level_001`–`level_005` asset patterns into schema v1; document M1 minimum element set; defer full GP3 catalogue and levels 006–020 explicitly.
