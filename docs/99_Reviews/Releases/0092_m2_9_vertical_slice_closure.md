## Motivation

**Why:** The product loop was incomplete — `recordCompletion` existed but was never called; wins stayed on an in-game terminal overlay; progress did not persist; Victory route was unused.

**Problem solved:** Player can complete a level, see celebration, land on Victory, and unlock the next level with save surviving app restart.

**Approach:** `CompletionSyncService` abstraction (local today, Firebase later) runs parallel to a minimum celebration duration (`victoryCelebrationMinDuration` 1.5s) so future remote sync fits without UX rework. `GameplayRewardMapper` bridges engine and data `RewardResult` types.
