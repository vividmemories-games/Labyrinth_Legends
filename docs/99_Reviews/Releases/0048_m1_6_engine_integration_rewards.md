## Task Summary

Completed M1 Core Engine Foundation by implementing stateless `RewardCalculator` with `RewardResult`, integrating reward calculation into `GameplaySession` on terminal `won`, and proving the full confirm → execute → evaluate → reward pipeline via `level_001.json` end-to-end and deterministic replay tests. **Out of scope:** UI/provider rewrite, save/progress, navigation, analytics, monetization, new mechanics.

## Motivation

**Why:** M1.5 determined win/loss but did not assign stars. M1.6 closes the Pure Dart engine loop and proves deterministic behaviour on the first authored level.

**Problem solved:** No star calculation from terminal session; no end-to-end engine proof for `level_001`.

**Approach:** `RewardCalculator.calculateFromTerminalSession()` reads won-phase facts only (path node count, gems collected); `GameplaySession` orchestrates after `ObjectiveEvaluator` transitions to `won`.

## Reward Calculation Behaviour

### When calculator runs

Only when `GameplaySession` transitions to `GameplayPhase.won` — immediately after objective evaluation succeeds.

### Inputs (terminal session state only)

| Fact | Source |
|------|--------|
| Path node count | `confirmedPath.length` per Level_Format §14 |
| Gems collected | `attemptContext.gemsCollected` |
| Total gems | `level.grid.countGems()` |
| Star thresholds | `level.starThresholds` |

**Does not** re-evaluate objectives, hazards, or path validity.

### Star rules (Level_Format §14)

| Path node count | Stars |
|-----------------|-------|
| ≤ `threeStars` | 3 |
| ≤ `twoStars` | 2 |
| ≤ `oneStar` | 1 |
| > `oneStar` | 0 |

Gem bonus: **deferred** — M1 uses path-node thresholds only per Level_Format §14 (Codex follow-up).
