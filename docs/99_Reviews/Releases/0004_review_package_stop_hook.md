## Task Summary

Added a Cursor **stop hook** that reminds the agent to scaffold and complete a review package when `lib/` or `test/` Dart files are newer than the latest review package in `docs/99_Reviews/`. Completes the automation started in review `0003` (generator script + workflow docs). No Flutter UI or gameplay changes.

**Out of scope:** `beforeSubmitPrompt` hooks, auto-filling review package content, CI integration, gameplay code.

## Motivation

Review `0003` introduced the generator script and mandatory review workflow, but agents could still finish a session without creating a review package. A **stop hook** closes that gap by automatically nudging the agent at session end when Dart code changed but no review package was updated.

This approach was chosen over heavier automation (CI gates, pre-commit hooks) because it is lightweight, project-local, and fits the existing Cursor agent workflow without blocking developers.

## Known Issues

- Hook uses file mtimes, not git diff — touching a Dart file without meaningful changes can trigger a reminder (low risk; agent can skip for minor fixes)
- `loop_limit: 1` prevents infinite follow-up loops but allows one reminder per session stop
- Hook does not detect doc-only major tasks (no Dart changes) — agent rules still cover that case
