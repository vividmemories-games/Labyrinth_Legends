## Task Summary

Created `docs/07_UI/UI_04_Design_Tokens.md` as the semantic design-token architecture document for the approved `docs/07_UI/` stack. The document defines token philosophy, token categories, semantic application rules across color, typography, spacing, shape, depth, motion, and iconography, gameplay-screen token needs, naming principles, usage rules, anti-patterns, success criteria, and governance boundaries. Supporting updates were made to `docs/07_UI/README.md` and `docs/07_UI/UI_00_Design_Principles.md` so the `07_UI` sequence reflects that `UI_04` is now authored. Out of scope: code changes, gameplay changes, widget APIs, component implementation, and canonical raw token-value changes.

## Motivation

**Why:** The approved `docs/07_UI/` stack needed a token-architecture layer between gameplay-screen behavior and the future component library so downstream UI work can express styling decisions semantically instead of inventing one-off visual rules.

**Problem solved:** Without a dedicated UI token-architecture document, later component, animation, responsive, and asset documents would have to infer token usage directly from mixed upstream sources, increasing the risk of raw-value leakage, parallel naming systems, and inconsistent gameplay emphasis.

**Approach:** Authored a documentation-only architecture layer downstream of LLDL, the canonical token catalog, and the approved UI architecture stack. Kept canonical values upstream in `docs/02_Design_System/Design_Tokens.md` and used `UI_04` to define how future UI documents should apply and extend semantic token meaning without overriding the upstream catalog.
